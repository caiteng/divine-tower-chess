# 游戏设计草案（v0.3 重构）

## 项目类型
单机塔防 + 自走棋升星原型。战斗从“固定路线/固定格子”重构为“连续二维空间自动战斗”。

## 核心战斗模型
- 战场是逻辑 2D 区域（默认宽 1000，高 800）。
- 单位与敌人都拥有连续坐标 `(x, y)`，并按速度连续移动。
- `x` 代表推进方向（敌右->左压水晶），`y` 代表纵深走位。
- 使用半径机制而非格子索敌：
  - `detectionRange`：感知范围
  - `attackRange`：攻击/治疗范围
  - `moveSpeed`：移动速度
- 行为循环：搜敌/搜友 -> 追近或拉扯 -> 进入范围后攻击/治疗 -> 目标丢失重选。

## 单位行为
- 近战：主动追击并贴近输出。
- 远程：优先在攻击距离外少量前压，到射程后输出。
- 治疗：搜索低血量友军，移动到治疗范围内治疗。
- 法师：以目标点为中心对局部半径造成AOE。

## 敌人与推进
- 敌人从右侧连续刷出并向左侧水晶推进。
- 优先攻击检测范围内最近友军，否则持续压向水晶。
- 第一版使用“直线推进 + 轻量分离(anti-overlap)”简化避让。

## 水晶与胜负
- 左侧有我方水晶。
- 敌人贴近水晶后造成水晶伤害。
- 水晶归零失败；当前波敌人清空后进入下一回合。

## 回合与养成规则（保留）
- 回合开始自动刷新商店（3格）。
- 购买、手动刷新、摆放、移动、开战都只允许准备阶段。
- 三合一升星：3个1星->2星，3个2星->3星。
- 3星后不再普通升星，只能走神品任务进阶。
- 升星候选可来自备战区+已上阵。
- 有神品任务的实例不能被普通升星消耗。

## 神品任务（实例绑定）
- 每个 3星且未神品化、未持有任务的实例：每回合开始独立 10% 判定。
- 可同时存在多个任务实例。
- 单实例最多1个进行中任务。
- 当前任务：
  - 战士 -> 狂战士：累计击杀1000
  - 牧师 -> 光法师：累计真实治疗100000
- 真实治疗仅统计实际恢复值，溢出/满血不计。

## 准备阶段摆放
- 当前实现为左侧“部署锚点”摆放（内部仍给出 lane/tile 索引用于兼容 UI）。
- 进入战斗后单位不再锁在锚点，会在连续空间自由移动。
- 重摆放必须保留：`instanceId/star/unitId/assignedTaskId/divine progress/isDivine`。

## 第一版目标
- 打通“主菜单 -> 准备 -> 2D自动战斗 -> 下一波/胜负”完整流程。
- 新手难度10波可手动游玩。
- 优先可玩性，不追求高成本特效。
