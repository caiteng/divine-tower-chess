# 游戏设计文档（统一版 v0.3）

## 1. 项目定义

这是一个 **2D 横版 / 伪纵深自动战斗原型**：
- 单位与敌人在二维连续空间中活动。
- 保留回合制准备阶段与成长系统（商店、升星、神品任务）。
- 保留波次推进与水晶防守。
- 明确禁止回退到 fixed lane / fixed route / tile board 的战斗与部署模型。

---

## 2. 战场与空间模型

- 战场是逻辑二维区域（width/height）。
- `x` 表示左右推进方向（敌人从右向左压水晶）。
- `y` 表示纵深位置。
- 我方部署区域位于战场左侧。
- 敌方出生区域位于战场右侧。
- 部署支持 deployment anchors（部署候选点），仅用于准备阶段落位，不表示棋盘格路径。

---

## 3. 回合结构

### 3.1 准备阶段（Preparation）
允许行为：
- 商店自动刷新（每回合开始）
- 手动刷新商店
- 购买单位
- 部署单位（bench -> placed）
- 调位（placed -> placed）
- 开始战斗

约束：
- 以上操作仅准备阶段可执行。
- 失败购买不应移除商店条目。
- 调位必须保留单位实例身份与全部成长状态。

### 3.2 战斗阶段（Battle）
单位行为基于半径机制：
1. 按 `detectionRange` 索敌（或治疗目标）
2. 若超出 `attackRange`，按 `moveSpeed` 移动接近/保持距离
3. 进入攻击/治疗范围后执行动作
4. 目标丢失时重新选择

敌人行为：
- 从右侧出生区域生成并向左推进。
- 有可攻击目标则战斗，无目标则继续压向水晶。

胜负：
- 敌人接触水晶造成水晶伤害。
- 水晶 HP 归零则失败。
- 当前波敌人清空后进入下一回合；最终波后胜利。

### 3.3 成长阶段（Growth）
- 单位成长通过三合一升星：
  - 3x1星 -> 1x2星
  - 3x2星 -> 1x3星
- 3星单位不再普通升星，只能通过神品任务进阶。

---

## 4. 神品任务（实例级规则）

- 任务分配按单位实例进行，不是全局或职业共享。
- 每个满足条件的 3星非神品实例，每回合开始独立进行 10% 判定。
- 多个实例可同时持有任务。
- 单实例同时最多 1 个进行中任务。
- 任务进度绑定 `unitInstanceId` 并在单局中跨回合累计。
- 当前任务：
  - Warrior -> Berserker：累计击杀 1000
  - Priest -> Light Mage：累计真实治疗 100000
- 治疗任务仅统计实际恢复值（不计溢出、不计满血目标）。

---

## 5. v0.3 实施边界

本轮只做“方向统一与旧抽象清理”：
- 不新增职业
- 不新增怪物
- 不新增神品任务种类
- 不引入真3D
- 不做复杂寻路
- 不做高成本美术/动画

目标是：让文档、类型、接口、出怪、部署模型全部服务同一套二维连续战斗设计。
