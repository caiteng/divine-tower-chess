# 游戏设计草案

## 项目类型
单机塔防原型游戏，结合自走棋式升星和神品进阶机制。

## 核心玩法
玩家每回合通过商店购买棋子，摆放在地图上的固定位置，抵御怪物沿路径进攻水晶。
击杀怪物可获得金币，用于购买棋子和刷新商店。
相同棋子可通过 3 合 1 合成为更高星级：
- 3个1星 -> 1个2星
- 3个2星 -> 1个3星
- 升星候选可以来自备战区和已上阵区域
- 若合并材料中有已上阵棋子，优先保留已上阵棋子作为升星后的主体，并保留其位置
- 3星棋子不能继续普通合并，只能通过神品任务升阶
- 已持有神品任务的棋子不会被普通升星消耗

3星为普通最高星级。
当棋子达到3星后，每回合开始时会按**棋子实例**进行10%独立判定：
- 不是“全局只发1个任务”
- 不是“同职业只发1个任务”
- 每个达到3星、且尚未神品化、且未接任务的实例都可独立触发
- 可同时存在多个神品任务
- 同一个棋子实例同时最多持有1个未完成任务

当前版本实现两种神品进阶：
1. 战士 -> 狂战士：该实例累计击杀1000个怪物后进阶
2. 牧师 -> 光法师：该实例累计真实治疗100000生命后进阶

任务进度规则：
- 与 `unitInstanceId` 绑定，不与职业类型绑定
- 同一局游戏内跨回合累积
- 已接任务实例在任务完成前不会重复接取第二个任务
- 神品单位不会再次接取神品任务
- 完成后对应实例自动进阶为目标神品单位

## 位置调整玩法
准备阶段允许玩家移动已上阵棋子位置：
- `movePlacedUnit(instanceId, lane, tileIndex)`
- 移动后只改变 `lane / tileIndex`
- 移动后保留实例身份与状态：`instanceId / unitId / star / assignedTaskId / divine progress / isDivine`
- 用于优化击杀或治疗任务效率

## 治疗统计规则
牧师/光法的任务进度只统计**真实治疗量**：
- 仅当目标友军当前生命 < 最大生命，且实际恢复成功时计入
- 只计入 `min(治疗量, 目标已损失生命值)`
- 满血目标不会累积任务进度
- 溢出治疗不会累积任务进度
- 禁止纯时间累计的伪治疗数值

## 棋子死亡规则
棋子在战斗回合中生命降至0时进入本回合死亡状态：
- 死亡棋子当前回合失效，不能继续攻击或治疗
- 死亡棋子不会被治疗在当前回合拉起
- 回合结束进入下一次准备阶段时，死亡棋子恢复到最大生命值
- 只重置死亡棋子的生命状态，仍存活但受伤的棋子不会被该规则自动补满

## 回合流程
1. 回合开始
2. 自动刷新商店，展示3个棋子
3. 对所有符合条件的3星实例独立判定神品任务
4. 玩家可购买棋子、刷新商店、摆放棋子、移动已上阵棋子、完成升星
5. 进入战斗阶段
6. 怪物沿路径进攻水晶，并对前排单位施加持续压力
7. 棋子自动攻击与施法，治疗按真实恢复量统计
8. 棋子生命归零后本回合失效
9. 清空当前波次后进入下一回合，死亡棋子在准备阶段恢复生命
10. 水晶生命归零则失败

## 阶段限制与失败回滚
- 手动刷新商店、购买、摆放、移动、开始战斗只能在准备阶段执行
- 战斗阶段不能继续购买、摆放、移动或重复开战
- 金币不足导致购买失败时，商店格子不会被移除
- 非法槽位、非法格子、已占用格子都返回失败，不改变现有状态

## 难度
- 新手：10波
- 普通：30波
- 困难：60波

## 地图
第一版仅实现1张地图，带1到2条固定扭曲路线。

## 第一版棋子
- 弓箭手
- 圣骑士
- 盾卫
- 战士
- 法师
- 牧师
- 骑兵
- 枪兵

## 神品棋子
- 狂战士（由战士进阶）
- 光法师（由牧师进阶）

## 第一版目标
做出完整可玩的原型：
- 可选择难度
- 可招募棋子
- 可摆放棋子
- 可调整已上阵棋子位置
- 可自动战斗
- 可升星
- 可触发并完成神品进阶任务
- 可完成一局游戏流程

## 非目标
- 不追求高质量美术
- 不追求完整平衡
- 不追求无bug
- 不实现联网、排行榜、广告、装备系统、复杂羁绊
